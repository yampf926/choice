# 월야고등학교: 침묵의 기록

`월야고등학교: 침묵의 기록`은 Java Swing으로 만든 선택형 스토리 게임입니다.  
플레이어는 학교에서 벌어진 연속 사고를 조사하면서 장면을 읽고, 선택지를 고르고, 단서를 모아 사건의 진실에 가까워집니다.

이 문서는 비전공자도 이해할 수 있도록 게임이 어떤 구조로 만들어졌는지, 화면이 어떻게 구성되는지, 어떤 방식으로 이야기가 진행되는지를 설명합니다.

## 1. 이 게임은 어떤 게임인가요?

이 게임은 소설을 읽듯이 진행되는 조사형 스토리 게임입니다.

- 화면에 장면 설명이나 인물 대사가 한 페이지씩 나타납니다.
- 플레이어는 클릭하거나 스페이스바를 눌러 다음 문장으로 넘어갑니다.
- 장면이 끝나면 선택지가 나타나고, 어떤 선택을 하느냐에 따라 다음 장면이 달라집니다.
- 조사 결과에 따라 진실에 가까운 결말, 잘못된 판단으로 이어지는 결말 등 여러 엔딩을 볼 수 있습니다.

쉽게 말해,  
"그림과 대사가 있는 인터랙티브 추리 소설"에 가깝습니다.

## 2. 어떤 기술로 만들었나요?

이 프로젝트는 **Java Swing**으로 만들어졌습니다.

Swing은 자바에서 창, 버튼, 글자, 패널 같은 화면 요소를 만들 때 쓰는 기본 UI 도구입니다.  
웹사이트처럼 브라우저에서 실행되는 방식이 아니라, 컴퓨터에서 직접 실행되는 데스크톱 프로그램 형태입니다.

이 프로젝트의 특징은 다음과 같습니다.

- 게임 전체가 거의 `src/Main.java` 한 파일 안에 들어 있습니다.
- 장면 데이터는 `assets/scenes.json`에 들어 있습니다.
- 배경/인물 이미지는 `assets/images/` 폴더에 있습니다.
- 저장 데이터는 프로젝트 최상위의 `save.properties` 파일에 저장됩니다.

## 3. 화면은 어떻게 생겼나요?

이 게임 화면은 크게 4개 층으로 생각하면 이해하기 쉽습니다.

### 3-1. 상단 바

화면 맨 위에는 현재 장면의 제목과 챕터 이름이 나옵니다.

- 왼쪽: 장면 제목
- 오른쪽: 챕터 표시

예를 들어 플레이어가 지금 어떤 사건을 조사 중인지, 어느 구간에 있는지를 알려주는 역할입니다.

### 3-2. 장면 배경 영역

화면 가운데 가장 큰 영역에는 장면 이미지가 보입니다.

이 영역 안에는 두 가지 그림이 겹쳐집니다.

- 배경 이미지
- 인물 이미지

배경은 교실, 복도, 옥상, 수영장 같은 장소를 보여주고,  
인물 이미지는 현재 장면의 화자나 등장 인물을 시각적으로 보여줍니다.

### 3-3. 하단 대화창

화면 아래쪽에는 반투명한 카드 형태의 대화창이 있습니다.

여기에는 다음 정보가 들어갑니다.

- 본문 텍스트
- 화자 이름표

대사는 한 글자씩 타이핑되는 느낌으로 나타나고,  
플레이어가 클릭하면 남은 문장이 즉시 전부 표시됩니다.

### 3-4. 오버레이 화면

필요할 때만 위에 덮어씌우는 화면도 있습니다.

- 시작 화면
- 엔딩 모음 화면
- 선택지 화면

이런 화면은 평소에는 숨겨져 있다가, 특정 상황에서만 나타납니다.

## 4. UX/UI는 어떤 식으로 구성했나요?

이 게임의 UX/UI는 "모바일 세로형 비주얼 노벨"에 가깝게 설계되어 있습니다.

### 4-1. 세로 비율 중심

게임 창은 `9:16`에 가까운 세로 비율을 유지하도록 맞춰져 있습니다.  
즉, 넓은 가로형 화면보다 스마트폰처럼 세로로 긴 구성을 기준으로 디자인되어 있습니다.

이 방식의 장점은 다음과 같습니다.

- 배경과 캐릭터를 세로로 길게 보여주기 좋음
- 하단 대화창을 안정적으로 배치하기 좋음
- 이야기 중심 게임에 어울리는 집중감 있는 화면 연출이 가능함

### 4-2. 레이어 방식

이 게임은 화면을 한 번에 그리는 대신, 여러 겹의 판을 쌓는 방식으로 구성되어 있습니다.

순서는 대략 이렇습니다.

1. 가장 아래에 배경 이미지
2. 그 위에 인물 이미지
3. 그 위에 어두운 오버레이 효과
4. 화면 아래에 대화창
5. 필요하면 그 위에 선택지나 시작 화면

이 구조 덕분에 장면 분위기를 유지하면서도, 필요한 UI를 유연하게 올릴 수 있습니다.

### 4-3. 어두운 톤 중심 디자인

게임 분위기는 미스터리, 오컬트, 조사극에 맞춰 어둡고 차분하게 잡혀 있습니다.

- 배경색은 짙은 남색/회색 계열
- 패널은 반투명하고 살짝 빛나는 느낌
- 강조색은 차가운 회청색 계열

이런 색 구성 덕분에 공포 게임처럼 과하게 자극적이지 않으면서도,  
조사물 특유의 긴장감과 침묵감을 유지합니다.

### 4-4. 버튼과 선택지

선택지는 카드형 버튼으로 보입니다.

- 짧은 텍스트는 2열로 배치
- 긴 텍스트는 1열로 넓게 배치
- 이미 완료한 조사 항목은 비활성화된 색으로 처리

즉, 플레이어가 "지금 누를 수 있는 선택"과 "이미 본 선택"을 시각적으로 구분할 수 있게 해 둔 구조입니다.

## 5. 이야기 진행은 어떻게 이루어지나요?

게임은 `장면(Scene)` 단위로 진행됩니다.

장면 하나에는 보통 이런 정보가 들어 있습니다.

- 장면 제목
- 챕터 이름
- 내레이션
- 화자
- 대사
- 배경 이미지
- 인물 이미지
- 선택지
- 장면 진입 시 실행할 상태 변화

플레이 방식은 다음 순서로 흘러갑니다.

1. 장면이 열립니다.
2. 배경과 인물 이미지가 갱신됩니다.
3. 텍스트가 페이지 단위로 표시됩니다.
4. 마지막 페이지까지 읽으면 선택지가 나타납니다.
5. 선택지에 따라 다음 장면으로 이동합니다.

즉, 이 게임은 "장면의 연속"으로 구성된 구조입니다.

## 6. 장면 데이터는 어디에 있나요?

실제 스토리 내용은 주로 `assets/scenes.json`에 들어 있습니다.

이 파일에는 각 장면의 내용이 JSON 형식으로 정리되어 있습니다.  
JSON은 사람이 읽고 쓰기 쉬운 데이터 형식이라고 생각하면 됩니다.

장면 데이터에는 예를 들어 이런 내용이 들어갑니다.

- 어떤 텍스트를 보여줄지
- 누가 말하는지
- 어떤 배경을 쓸지
- 어떤 인물 이미지를 쓸지
- 다음으로 갈 수 있는 선택지가 무엇인지

개발자는 자바 코드 안에 이야기를 하드코딩하기보다,  
이 JSON 파일을 읽어 장면을 만들어 화면에 뿌리는 방식으로 작업하고 있습니다.

## 7. 이미지는 어떻게 표시되나요?

이미지는 `assets/images/` 폴더에 있습니다.

종류는 크게 두 가지입니다.

- `bg_...`: 배경 이미지
- `ch_...`: 캐릭터 이미지

표시 방식도 다릅니다.

- 배경 이미지는 화면을 가득 채우는 방식
- 캐릭터 이미지는 비율을 유지한 채 화면 안에 맞춰 배치하는 방식

덕분에 배경은 장소를 넓게 보여주고,  
인물은 찌그러지지 않으면서 자연스럽게 중앙에 배치됩니다.

## 8. 텍스트는 왜 페이지처럼 나뉘나요?

대화창은 높이가 정해져 있기 때문에, 긴 문장을 한 번에 전부 넣으면 아래가 잘릴 수 있습니다.  
그래서 게임은 긴 문장을 자동으로 적당한 길이로 잘라 여러 페이지로 나눕니다.

이 과정에서 고려하는 요소는 다음과 같습니다.

- 한 페이지에 들어갈 수 있는 줄 수
- 현재 폰트 크기
- 줄바꿈이 어디에서 일어나는지
- 문장 부호 근처에서 자연스럽게 끊을 수 있는지

즉, 텍스트가 그냥 기계적으로 잘리는 것이 아니라,  
읽기 편한 위치에서 나뉘도록 설계되어 있습니다.

## 9. 저장은 어떻게 되나요?

이 게임은 진행 상황을 `save.properties` 파일에 저장합니다.

저장되는 정보 예시는 다음과 같습니다.

- 플레이어 이름
- 현재 장면 ID
- 텍스트 속도
- 각 사건 해결 여부
- 진실 점수 / 의심 점수
- 본 엔딩 목록
- 획득한 단서

그래서 게임을 껐다가 다시 켜도,  
`이어하기` 버튼으로 마지막 진행 지점에서 다시 시작할 수 있습니다.

## 10. 시작 화면은 어떻게 구성되나요?

시작 화면은 게임 장면 위에 카드 하나를 띄우는 방식입니다.

들어 있는 요소는 다음과 같습니다.

- 게임 제목
- 이름 입력칸
- 시작 버튼
- 이어하기 버튼
- 엔딩 모음 버튼
- 기록 삭제 버튼

이 화면도 별도 창을 띄우는 것이 아니라,  
기존 장면 영역 위에 오버레이 카드로 덮는 방식이라 전체 스타일이 통일됩니다.

## 11. 엔딩 모음은 무엇인가요?

엔딩 모음은 플레이어가 본 결말을 따로 정리해서 보여주는 화면입니다.

- 이미 본 엔딩은 제목과 설명을 표시
- 아직 못 본 엔딩은 `???`처럼 숨김 처리

이 기능 덕분에 플레이어는 "아직 못 본 결말이 남아 있구나"를 쉽게 알 수 있습니다.

## 12. 개발 관점에서 이 프로젝트의 특징은?

이 프로젝트는 구조가 단순한 대신, 한 파일에 많은 역할이 모여 있습니다.

장점:

- 빠르게 수정 가능
- 작은 프로젝트에서 흐름 파악이 쉬움
- 장면 추가와 UI 수정이 한 파일 안에서 연결되어 보임

주의할 점:

- 기능이 많아질수록 `Main.java`가 길어짐
- UI, 상태, 저장, 스토리 로딩이 한곳에 모여 있어 관리가 어려워질 수 있음

즉, 현재는 "작은 게임을 빠르게 만들기 좋은 구조"에 가깝습니다.

## 13. 파일 구조는 어떻게 보면 되나요?

대략 이렇게 이해하면 됩니다.

```text
choice/
├─ src/
│  └─ Main.java               # 게임 실행, UI, 상태, 저장, 렌더링의 중심
├─ assets/
│  ├─ scenes.json             # 스토리 장면 데이터
│  └─ images/                 # 배경/캐릭터 이미지 파일
├─ out/production/choice/     # 컴파일된 클래스 파일 위치
├─ save.properties            # 저장 데이터
└─ README.md                  # 프로젝트 설명 문서
```

## 14. 실행 흐름을 아주 간단히 요약하면

이 게임은 아래 순서로 움직입니다.

1. 프로그램 실행
2. `Main.java`가 창을 만듦
3. `scenes.json`을 읽어 장면 목록 로드
4. 시작 화면 표시
5. 플레이어가 시작 또는 이어하기 선택
6. 장면 표시
7. 텍스트 진행
8. 선택지 표시
9. 상태 저장
10. 엔딩 도달 시 엔딩 모음에 반영

## 15. 한 줄 요약

이 프로젝트는  
**Java Swing으로 만든 세로형 선택형 스토리 게임**이고,  
**배경 이미지 + 인물 이미지 + 하단 대화창 + 오버레이 선택지** 구조로 화면을 구성하며,  
**`scenes.json`의 장면 데이터를 읽어 조사형 스토리를 진행하는 방식**으로 동작합니다.
